Content Management System (WCMS)

Website managment tool

The Web Content Management System (WCMS) is Indiana University's website managment tool built from Hannon Hill's Cascade CMS. It is one of the main tools used at DUE as it is what is used to create, edit, publish and manage the over 30 individual websites overseen by the office.

WCMS was something new coming from studying and writing code ourselves as the system is more form based, similar to something like Wordpress. In WCMS each page is built with some settings, like display name, navigation, or banner, then sections with chunks inside of those sections. There are many different kinds of chunks like text, images, accordions, videos, feeds, etc.

I found it very interesting how websites were set-up here and I've began using it in my own websites. It is very useful to know how to manage websites on such a large scale for something as big as a university.





Salesforce Marketing Cloud

Email communications creation and sending

Salesforce is the system used to not only build, but send mass communication emails out as well. I had not built emails prior to learning salesforce here, but I quickly enjoyed the process. I was surprised to learn they can be built just like any website. Emails are made with an IU Framework, using many pre-built chunks designed to work for all devices which are able to be pieced together and added to.

Service Now

Ticketing system

Servicenow is used to manage any requests sent to DUE Communications, referred to as tickets. These include requests for the communications side, typically assigned to DUE.COMM, and web support tickets, typically assigned to DUE.WEB. Web support tickets can include many various things from uploading/updating digital signage, making updates to a webpage or a person's profile, managing the access to certain people, uploading videos for captioning, adding or removing pages/content on websites or even sending out an email.





Basecamp

Projects and documentation management

Basecamp is an extremely helpful tool as it performs many functions which help keep not only myself, but the whole office organized. The two main features I use most frequently include viewing any upcoming project assignments I need to complete and searching the tons of documentation created by the office for any questions I may have.

Basecamp has individual tabs for each project created where everyone working on the specific project and the files for it can be added. The most common projects I recieve here are creating an email in Salesforce or build the page for a news story created by one of the Content Interns. The most helpful feature here is being able to break-down everthing into individual to-dos and assigning them to people with due dates which are displayed on an 'upcoming assignments' for that person. Because of Basecamp, I always know what I need to be working on and I never feel lost.

These individual project tabs are not only used for each project but they also hold the main spaces for the web and communications teams in the office. In these are many folders of documentation for everything the office does which is extremely helpful for any task that may require a refresher, especially the ones done not super frequently like uploading videos or accessing the internal files, like .htaccess, of a website.
